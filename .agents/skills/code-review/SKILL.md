# Code Review Skill

When reviewing code, follow these steps to ensure technical excellence and maintainability.

## Review Checklist

1.  **Correctness**: Does the code perform the intended logic without side effects?
2.  **Edge Cases**: Are potential error conditions, null values, and boundary cases handled?
3.  **Style & Conventions**: Does the code adhere to the project's style guide and SOLID principles?
4.  **Performance**: Are there any obvious inefficiencies, such as unnecessary loops or memory leaks?
5.  **Security**: Check for common vulnerabilities like hardcoded secrets or improper input sanitization.

## Guidance for Feedback
* Provide constructive, actionable comments.
* Suggest specific code improvements or refactors where applicable.
* Prioritize issues based on severity (e.g., Critical, Minor, Nitpick).
