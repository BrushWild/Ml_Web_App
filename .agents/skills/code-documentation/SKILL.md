# Code Documentation Skill

This skill ensures that all code is accessible and maintainable through high-quality documentation.

## Documentation Standards

1.  **README Files**: Maintain a clear project overview, installation steps, and usage examples.
2.  **API Specifications**: Ensure all endpoints, parameters, and response types are documented using OpenAPI/Swagger standards.
3.  **Inline Comments**: Use JSDoc, Doxygen, or language-specific equivalents for complex logic, while striving for self-documenting code.
4.  **Architecture Diagrams**: Use Mermaid.js syntax to describe system flows and component relationships.

## Best Practices
* Write for the target audience (e.g., developers vs. stakeholders).
* Keep documentation "living" by updating it alongside code changes.
* Ensure all setup instructions are tested and reproducible.
